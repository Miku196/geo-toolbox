## 触发条件

用户提及以下任一关键词或意图时激活：
- 地理空间分析 / GIS 处理 / 遥感分析 / 碳核算
- 流域 / 生态修复 / 地质灾害 / 海岸带 / 能源选址
- NDVI / RUSLE / SCS-CN / InVEST / Mann-Kendall
- 数据搜索 / STAC / OSM / GEE / 矢栅转换
- "geo-toolbox" / "geo 分析" / "跑一下 geo"
- 任何涉及需要多个 GIS 步骤组合完成的综合性任务

## 输入参数

| 参数 | 类型 | 必填 | 描述 |
|------|------|------|------|
| `workspace` | string | ✅ | 工作根目录 |
| `domain` | string | 🟡 | 分析领域（ecology/hydro/geohazard/coastal/carbon/energy/forestry/agri/urban/survey/climate），多领域用逗号分隔 |
| `boundary` | string | 🟡 | 研究区矢量边界（GeoJSON 或 Shapefile 路径） |
| `aoi_name` | string | 🟡 | 研究区名称 |
| `year` | int | 🟡 | 分析年份 |
| `skip_stages` | string | ❌ | 逗号分隔要跳过的阶段，如 `"stage0,stage3"` |

注：🟡 表示根据 `domain` 不同可能必填

```
{{workspace}}/raw/            ← Stage 0: 原始下载数据
{{workspace}}/prepared/       ← Stage 1: 标准化数据
{{workspace}}/intermediate/   ← 各阶段中间结果
```

## 四专家分工体系

| 专家 | 角色 | 对应工具组 |
|------|------|------------|
| **Finder** | 数据发现与下载 | stac_search, osm_query_bbox, duckdb_query, geohash_*, h3_*, tile_latlon_to_tile |
| **Core** | 数据标准化与基础处理 | crs_list, crs_transform, vector_*, temporal_trend, zonal_stats, morans_i, gistar, idw_grid, jenks_classify, report_carbon, report_render |
| **Plugin** | 领域模型计算 | carbon_*, ecology_*, hydro_*, geohazard_*, coastal_*, energy_*, forestry_*, agri_*, urban_*, survey_*, climate_* |
| **Adapter** | 外部生态集成 | stac_search, osm_query_bbox, duckdb_*, ingest_*, postgis_*, gee_*, qgis_*, cad_* |

## 工作流阶段（按 domain 自适应）

### Stage 0 — 数据发现与下载（Finder）

> 按需启动。若用户已提供数据，跳过。

**分支 A: 卫星影像搜索 (STAC)**

```bash
# 获取研究区边界
BOUNDARY_JSON=$(cat "{{boundary}}")
MIN_LON=$(echo "$BOUNDARY_JSON" | jq '.bbox[0]')
MIN_LAT=$(echo "$BOUNDARY_JSON" | jq '.bbox[1]')
MAX_LON=$(echo "$BOUNDARY_JSON" | jq '.bbox[2]')
MAX_LAT=$(echo "$BOUNDARY_JSON" | jq '.bbox[3]')

# STAC 搜索 Sentinel-2
RESULT=$(curl -s -X POST http://localhost:9378/api/call \
  -H "Content-Type: application/json" \
  -d "{
    \"tool\": \"stac_search\",
    \"args\": {
      \"collection\": \"sentinel-2-l2a\",
      \"min_lon\": $MIN_LON, \"min_lat\": $MIN_LAT,
      \"max_lon\": $MAX_LON, \"max_lat\": $MAX_LAT,
      \"date_from\": \"{{year}}-01-01\",
      \"date_to\": \"{{year}}-12-31\",
      \"limit\": 20
    }
  }")
echo "$RESULT" | jq . > "{{workspace}}/raw/stac_items.json"
```

**分支 B: OSM 路网/建筑/水系**

```bash
RESULT=$(curl -s -X POST http://localhost:9378/api/call \
  -H "Content-Type: application/json" \
  -d "{
    \"tool\": \"osm_query_bbox\",
    \"args\": {
      \"min_lon\": $MIN_LON, \"min_lat\": $MIN_LAT,
      \"max_lon\": $MAX_LON, \"max_lat\": $MAX_LAT,
      \"feature_type\": \"waterway\"
    }
  }")
echo "$RESULT" | jq . > "{{workspace}}/raw/osm_waterway.json"
```

**分支 C: 坐标转换（需要时）**

```bash
# CRS 列表
curl -s -X POST http://localhost:9378/api/call \
  -d '{"tool":"crs_list","args":{}}' | jq .

# 坐标转换
curl -s -X POST http://localhost:9378/api/call \
  -d '{"tool":"crs_transform","args":{"from_epsg":4326,"to_epsg":3857,"x":104.06,"y":30.67}}' | jq .
```

**分支 D: 高斯换带（测绘场景）**

```bash

curl -s -X POST http://localhost:9378/api/call \
  -d '{"tool":"survey_zone_info","args":{"lon":104.06}}' | jq .

# 高斯正算
curl -s -X POST http://localhost:9378/api/call \
  -d '{"tool":"survey_gauss_forward","args":{"b":30.67,"l":104.06,"l0":105,"ellipsoid":"CGCS2000"}}' | jq .
```

### Stage 1 — 数据标准化（Core / 串行）

**步骤 1.1: 矢量处理**

```bash
# 面积计算
curl -s -X POST http://localhost:9378/api/call \
  -d "{\"tool\":\"vector_area\",\"args\":{\"geojson\": $(cat {{boundary}})}}" | jq .

# 质心
curl -s -X POST http://localhost:9378/api/call \
  -d "{\"tool\":\"vector_centroid\",\"args\":{\"geojson\": $(cat {{boundary}})}}" | jq .

# 缓冲 (1000m)
curl -s -X POST http://localhost:9378/api/call \
  -d "{\"tool\":\"vector_buffer\",\"args\":{\"geojson\": $(cat {{boundary}}), \"distance_m\":1000}}" \
  | jq . > "{{workspace}}/prepared/buffer_1km.json"

# 裁剪
curl -s -X POST http://localhost:9378/api/call \
  -d "{\"tool\":\"vector_clip\",\"args\":{\"geojson_target\":..., \"geojson_clip\":$(cat {{boundary}})}}" | jq .
```

**步骤 1.2: 空间索引编码**


## 错误处理

1. **MCP Server 不可用**：所有工具调用前检查 `curl -s http://localhost:9378/health`，若失败提示启动 `geo-toolbox mcp-serve`
2. **命令失败**：每次 `curl` 后检查 `jq -e '.error'`，若有错误立即终止并报告
3. **Stage 跳过**：解析 `{{skip_stages}}` 跳转
4. **工具缺失**：若某 domain 的工具未注册（如未编译对应 feature），返回空结果即可
5. **参数校验**：`validate_coord` 工具可用于校验输入经纬度合法性
